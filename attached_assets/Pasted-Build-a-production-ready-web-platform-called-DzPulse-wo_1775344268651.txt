Build a production-ready web platform called “DzPulse” (working name), inspired by the browsing experience and card-based UI structure of Polymarket, but STRICTLY for public opinion polling and sentiment aggregation in Algeria only.

This is NOT a betting app, NOT a prediction market, and NOT a trading platform.
There must be:
- no money
- no wallets
- no crypto
- no buy/sell
- no odds
- no shares
- no portfolio
- no wagering language
- no gambling mechanics

The platform is a modern Algeria-only polling platform where users vote on public-interest questions and explore live sentiment across Algeria.

==================================================
1. PRODUCT GOAL
==================================================

Create a platform where Algerian users can:
- browse live polls
- vote once per poll
- see real-time results
- filter polls by category, status, region, popularity, and recency
- view poll detail pages
- discuss polls in comments
- submit poll suggestions
- see trending sentiment across topics in Algeria
- optionally view results by demographic and wilaya if data is available

The core UX should feel similar to Polymarket in structure:
- card-based browse page
- strong event/poll titles
- category tabs
- search
- trending sections
- detail pages per poll
- clean market-like layout

But the visual identity should be:
- white theme first
- brighter, cleaner, more editorial
- lighter than Polymarket
- minimalist, premium, sharp, modern
- same level of information density and clarity as Polymarket
- no dark gambling vibe

6. POLLING RULES
==================================================

Implement strict polling logic:
- one vote per user per poll
- guest mode may browse only
- if anonymous voting is allowed, use strong anti-spam logic
- prevent duplicate votes using:
  - authenticated account
  - IP hash
  - device/user-agent hash
  - rate limiting
- allow vote change only if explicitly enabled before poll closes
- closed polls become read-only
- upcoming polls display countdown
- show percentages and total votes after submission
- configurable visibility:
  - immediate results
  - delayed results
  - results only after vote
